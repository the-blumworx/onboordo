# Contributing to Onboordo

First off, thank you for considering contributing to Onboordo! It's people like you that make Onboordo such a great tool for the community.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [issue tracker](https://github.com/your-username/onboordo/issues) to see if someone else has already created an issue for it. If not, go ahead and make one!

## Contribution Workflow

We follow a standard Fork & Pull Request workflow. To contribute, please adhere to the following steps:

1. **Discuss Before Coding**: Please **create an issue** for discussion before opening a Pull Request. This ensures your work aligns with the project's roadmap and doesn't duplicate ongoing efforts.
2. **Fork the Repository**: Fork the project to your own GitHub account and clone it to your local machine.
3. **Create a Branch**: Create a new branch for your feature or bug fix (`git checkout -b feature/your-feature-name` or `bugfix/issue-number`).
4. **Make Your Changes**: Write your code.
5. **Include Unit Tests**: All new features and bug fixes **must include corresponding unit tests**. Ensure all tests pass.
6. **Run Linters and Formatters**: Ensure your code follows the project's style guidelines by running the required linter and formatter before committing your changes.
7. **Commit and Push**: Commit your changes with a descriptive message and push to your fork.
8. **Open a Pull Request**: Submit a Pull Request to the main repository.

## Getting Help

Join our Discord community or reach out via GitHub Issues if you have any questions or need guidance! We are happy to help you get your first PR merged.
